Tailwind

Tailwind CSS är ett CSS-ramverk med öppen källkod. Skillnaden med detta ramverk är att det till skillnad från andra CSS-ramverk så som Bootstrap, inte tillhandahåller en serie fördefinierade designer för element som knappar eller tabeller. Istället har Tailwind den en lista med "hjälpmedels"-CSS-klasser som kan användas för att styla varje element.
Till exempel, i andra traditionella system, skulle det finnas en klassmeddelandevarning som skulle tillämpa en grön bakgrundsfärg och fet text. För att uppnå detta resultat i Tailwind måste du använda en uppsättning klasser skapade av biblioteket: bg-green-200 och font-bold.
Tailwind har blivit mycket populärt senaste åren, speciellt i kombination med JavaScript-ramverken så som React och Vue.js. Tailwind kan även kompileras för att endast spara kod över de klasser man använder sig av. Länk till webbplatsen.

Fördelar

Efter att ha testat mig att använda av Tailwind tycker jag att det upplevs ganska lätt att lära sig. Även fast jag förstår att det finns tusentals olika funktioner att lära sig, får man snabbt in hur man använder sig av klasserna.

Det är också väldigt lätt att jobba med färgerna. Här på denna sida har jag använt mig av samma färg överallt, men endast ändrat på färgens ljus/mörker med hjälp av färgkodens mått.

Nackdelar

Jag upplever att det krävs väldigt mycket kod men framförallt väldigt mycket upprepning i Tailwind. Även fast ramverket hjälper till att skriva klasser (har även använt mig av pluginet Tailwind CSS IntelliSense) så tycker jag faktiskt det hade varit lättare att bygga en klass i en CSS-fil som jag sedan anropar.

Nu när jag byggde en tabell behövde jag lägga en lång klass på varje tr, istället för att ha skapat regler över alla tr som jag inte ens hade behövt anropa.
